CYCLES v2 Hardware

Hardware Requirements

Plain-language requirements for the v2 custom PCB. Capability-level, not chip-specific.

Prepared for EE consultation · 2026-04-28

CYCLES is a stereo audio looper pedal. v1 runs on the Daisy Seed dev board; v2 will move to a custom PCB. Sections 1–6 + 8 capture non-negotiable capabilities. Section 7 is a worksheet of decisions where EE input is wanted.

1 · USB Firmware Updates

End users update firmware by plugging a USB cable into the pedal and using a web browser. No extra software, no separate programmer.

The main chip must support this natively. Some cheaper chips can't — they need an extra helper chip on the board, or only update through a serial cable. Those aren't acceptable.

Connector + protection

  • USB-C connector, accessible on the enclosure
  • ESD protection on the data lines
  • Recovery path if firmware corrupts (recessed button, internal jumper, etc.)
  • USB 5 V rail isolated from audio analog rails
2 · No Wireless

No Bluetooth, no WiFi. Adds cost, certification, RF noise near audio, and pairing UX issues without solving a real user need. USB-C handles all file transfer.

3 · Power Input

Pedal ships without a wall wart. User supplies their own.

Spec on packaging + manual + enclosure label: "Regulated 9 V DC center-negative, 500 mA min, UL/CE-listed PSU recommended."

Defensive DC input design

  • Reverse polarity protection
  • Overvoltage clamp / TVS (in case user plugs in 18 V)
  • Inrush current limiting

A wrong or noisy PSU should damage itself, not the pedal.

4 · MIDI

Both physical and USB MIDI required.

Physical MIDI

  • Two 3.5 mm TRS jacks: one IN, one OUT/THRU
  • TRS MIDI Type A (MIDI Association ratified standard, 2018)
  • Standard input opto-isolation + output current-limiting per spec
  • Ship a TRS-to-5-pin-DIN adapter cable in the box

USB MIDI

  • Bidirectional in + out over the USB-C cable already required for firmware updates
  • No extra hardware
5 · Expression / Aux Combo Jack

One 3.5 mm TRS jack auto-detects expression pedal (continuous rocker) vs aux footswitch (1–2 momentary switches).

Firmware-assignable target — user picks via menu what the jack controls.

  • Empress-style detection circuit
  • Analog input pin for expression voltage; digital input + pull-up for aux switch
6 · Enclosure + I/O

Target enclosure: folded sheet metal (open to alternatives if there's a strong reason).

Total I/O — 8 jacks

  • Stereo audio IN L/R + stereo OUT L/R (4)
  • USB-C (1)
  • MIDI IN + OUT, 3.5 mm TRS (2)
  • EXP/AUX combo, 3.5 mm TRS (1)
  • DC power in (1)

Top face: 3 footswitches, 2 pots, 1 rotary encoder w/ switch, 7-segment 2-digit display, 8 RGB LEDs, 3 toggles.

PCB split is your call

Single board / main + jack daughterboard / top control board + bottom audio — propose based on enclosure dimensions.

7 · Component Worksheet

For each item: chip/part recommendation, alternatives, trade-offs, open questions.

Brain & memory
1.
Main chip (MCU)

Currently STM32H750 (Daisy Seed). Strong preference to stay on STM32H7-series for firmware portability.

2.
External SDRAM size

64 MB today on Daisy. With eMMC handling bulk loop storage (item 3), SDRAM only needs to be a working buffer for the audio engine.

3.
Loop storage — three-tier Decided

eMMC = live recording target (4–16 GB, sized for performance not capacity). MicroSD slot = user library + emergency backup mirror, removable. SDRAM (item 2) = audio engine working buffer. Raw NAND rejected (multi-week wear-leveling firmware not viable at our scale). EE input on: eMMC chip choice + size, microSD slot type/orientation.

8 · Certifications · US + Europe
  • FCC Part 15 Class B (US) — unintentional emissions
  • CE Mark (Europe) — EMC compliance
  • RoHS (Europe) — lead-free, no banned substances
  • REACH (Europe) — chemical disclosure
  • WEEE (Europe) — recycling scheme registration

Not needed

  • UL / CSA on the pedal itself (only the wall wart needs it — user-supplied)
  • FCC ID (no intentional radio)

Cert has to be redone if the PCB layout or enclosure changes significantly. Lock the design before testing.
