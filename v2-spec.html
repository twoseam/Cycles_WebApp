CYCLES v2 — Hardware Requirements

Plain-language requirements for the v2 custom PCB. Capability-level, not chip-specific.

Prepared for EE consultation · 2026-04-28

CYCLES is a stereo audio looper pedal. v1 runs on the Daisy Seed dev board; v2 will move to a custom PCB. This document captures non-negotiable capability requirements (sections 1–6 + 8) and a worksheet of component decisions where EE input is wanted (section 7).

Section 1

USB Firmware Updates

End users must be able to update firmware by plugging a USB cable into the pedal and using a web browser — no extra software, no separate programmer.

Implication for chip choice: the main chip must support this natively. Some cheaper chips can't — they require an extra helper chip on the board, or only support updates through a serial cable. Those are not acceptable.

Connector + protection

  • USB-C connector (not micro/mini), accessible on the enclosure
  • ESD protection on the USB data lines
  • A way to recover the pedal if firmware corrupts (recessed button, internal jumper, or equivalent)
  • USB 5 V rail kept isolated from audio analog rails

Section 2

No Wireless

No Bluetooth, no WiFi. Decided against — adds cost, certification work, RF noise near audio, and pairing UX issues without solving a real user need. USB-C handles all file transfer.

Section 3

Power Input

Pedal will be sold without a wall wart. User supplies their own power.

Required spec on packaging + manual + enclosure label: "Regulated 9 V DC center-negative, 500 mA min, UL/CE-listed PSU recommended."

EE must design defensively on the DC input

  • Reverse polarity protection
  • Overvoltage clamp / TVS (in case user plugs in 18 V by mistake)
  • Inrush current limiting

Goal: a wrong or noisy PSU damages itself, not the pedal.

Section 4

MIDI

Both physical and USB MIDI required.

Physical MIDI

  • Two 3.5 mm TRS jacks: one IN, one OUT/THRU
  • TRS MIDI Type A (MIDI Association ratified standard, 2018) — most common in modern pedals
  • Standard MIDI input opto-isolation + output current-limiting per spec
  • Ship a TRS-to-5-pin-DIN adapter cable in the box (kills compatibility complaints from users with legacy gear)

USB MIDI

  • Bidirectional in + out over the existing USB-C cable
  • No extra hardware — comes free with the USB connection already required for firmware updates
  • Lets the pedal talk to a DAW / phone / laptop without any extra cabling

MCU implication: any modern MCU has spare UART peripherals to handle MIDI. Not a constraint.

Section 5

Expression / Aux Combo Input

One 3.5 mm TRS jack that auto-detects whether an expression pedal (continuous rocker) or an aux footswitch (1–2 momentary switches) is connected.

Firmware-assignable target: user picks via menu what the jack controls — could be a built-in pedal parameter (mix, stretch, speed, volume, etc.) or a programmable target the user defines.

EE notes

  • Standard tip-ring-sleeve jack with detection circuit (Empress-style)
  • Analog input pin on MCU for expression voltage
  • Digital input + pull-up for aux switch detection
  • ~$1–2 in parts, minimal panel space

Section 6

Enclosure + PCB Layout

Target enclosure: folded sheet metal (TBD — open to EE input on alternatives like cast aluminum / Hammond-style if there's a strong reason).

Total I/O count (drives panel layout + board-split decision)

  • 4 audio jacks: stereo IN L/R + stereo OUT L/R
  • 1 USB-C
  • 2 MIDI: IN + OUT (3.5 mm TRS)
  • 1 EXP/AUX combo (3.5 mm TRS)
  • 1 DC power in
  • 8 jacks total

Plus on the top face: 3 footswitches, 2 pots, 1 rotary encoder w/ switch, 7-segment 2-digit display, 8 RGB LEDs, 3 toggles.

PCB split is EE's call

Common options for this much I/O:

  • Single board (cheapest, fewest failure points — only if enclosure size allows)
  • Main board + jack daughterboard (frees the main board from edge constraints)
  • Top control board + bottom audio board (good for short, wide enclosures)

EE to propose based on final enclosure dimensions.

Section 7

Component Worksheet

For each item below, please share what you're thinking — chip/part recommendations, alternatives you'd consider, trade-offs, and any open questions.

Brain & memory
1. Main chip (MCU)
Currently STM32H750 (Daisy Seed). Strong preference to stay on STM32H7-series for firmware portability.
2. External SDRAM size
64 MB today on Daisy. With eMMC handling bulk loop storage (item 3), SDRAM only needs to be a working buffer for the audio engine.
3. Loop storage — three-tier architectureDecided
  • eMMC is the live recording target (4–16 GB, sized for performance, not capacity). All real-time writes go here.
  • MicroSD card slot is for user library + emergency backup mirror of the live loop. User-removable for sneaker-net to laptop.
  • SDRAM (item 2) is the working buffer for the audio engine.
  • Raw NAND was considered and rejected — wear-leveling firmware is multi-week senior work and a fit only at much higher production scale.
EE input wanted on: eMMC chip choice + size, microSD slot type/orientation.

Section 8

Certifications (target markets: US + Europe)

  • FCC Part 15 Class B (US) — unintentional emissions test
  • CE Mark (Europe) — equivalent EMC compliance
  • RoHS (Europe) — lead-free, no banned substances. Spec accordingly.
  • REACH (Europe) — chemical disclosure paperwork
  • WEEE (Europe) — recycling scheme registration

Not needed (no AC mains, no radio)

  • UL / CSA on the pedal itself (only the wall wart needs it — user's responsibility since we don't ship one)
  • FCC ID (no intentional radio)

Lock the design before testing. Cert has to be redone if the PCB layout or enclosure changes significantly.
